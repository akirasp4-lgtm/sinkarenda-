# 【依頼】LINE で「倉庫作業」と送ったら、その日の倉庫作業内容を返す機能

依頼元: 予定管理アプリ作成プロジェクト
依頼先: GR社内秘書ボット(ラインボット)プロジェクト
日付: 2026-05-18

---

## 背景

予定管理アプリで「倉庫モード + 倉庫タスク(清掃 / 備品管理 / 工具メンテナンス / 入荷対応 / 出荷準備 / 廃材処理)」を選択して日報登録できるようになった。
これを LINE 経由で簡単に確認できるようにしたい。

社員が LINE で **「倉庫作業」** と送ったら、その日 倉庫で誰が何をしているかを返信する機能を追加してほしい。

---

## 期待する動作

```
[社員]
> 倉庫作業

[Bot]
今日の倉庫作業 (2026-05-18):

▼ グローライズ
・藤木さん(代表) → 清掃、備品管理
・田中さん → 入荷対応

(該当者なし日)
[Bot]
今日 (2026-05-18) は倉庫作業の予定がありません。
```

メッセージ書式は自由。読みやすければOK。

## バックエンド情報

### GAS エンドポイント

既に予定管理アプリ側で **デプロイ済み(v32 / 2026-05-18)** です。

```
URL: https://script.google.com/macros/s/AKfycbxp2eUcpIjCj0ZWyAPPD9m3egJrKdWmXRK2AVnFrmBm4iO1QHCk-FZEH5LFFv7OloqcjQ/exec
```

(既存の `EVENT_GAS_URL`= 同じ URL なので、新たに環境変数を増やす必要なし)

### 認証

既存の **`VEHICLE_RES_TOKEN`** (`車両予約用トークン1234`) を流用してください。
すでに `.env` に設定済みのはず。

### リクエスト

```http
POST URL HTTP/1.1
Content-Type: text/plain

{
  "action": "warehouse_today",
  "token": "車両予約用トークン1234",
  "date": "2026-05-18"
}
```

| フィールド | 必須 | 説明 |
|---|---|---|
| action | ✅ | 必ず `"warehouse_today"` |
| token | ✅ | 既存の `VEHICLE_RES_TOKEN` |
| date | ❌ | `YYYY-MM-DD` 省略時は今日の日付 |

### 成功レスポンス(例)

```json
{
  "status": "ok",
  "date": "2026-05-18",
  "entries": [
    {
      "name": "藤木",
      "role": "代表",
      "tasks": "清掃、備品管理",
      "company": "グローライズ",
      "id": "abc-1234-..."
    },
    {
      "name": "田中",
      "role": "同行",
      "tasks": "清掃、備品管理",
      "company": "グローライズ",
      "id": "abc-1234-..."
    }
  ],
  "grouped": [
    {
      "tasks": "清掃、備品管理",
      "company": "グローライズ",
      "leader": "藤木",
      "members": ["藤木", "田中"]
    }
  ]
}
```

### フィールド説明

- **entries**: 倉庫モード(夜勤カラム='倉庫')の **1 人 1 行** 形式。生データ
- **grouped**: 同じグループ ID で集約済み。`grouped` のほうが LINE 表示に向くと思う
  - `leader`: 代表者名(無ければ空)
  - `members`: 代表 + 同行 全員の名前配列
  - `tasks`: カンマ区切りのタスク文字列(例: "清掃、備品管理")
  - `company`: 会社名

### エラーレスポンス

```json
{"status":"error","message":"認証失敗"}     // トークン不一致
{"status":"error","message":"日報シートが見つかりません"}
```

## ラインボット側でやってほしい実装

### 1. メッセージハンドラに追加

社員が LINE で `倉庫作業` (完全一致 or 含む)を送ったら、新しいハンドラに分岐させる。

例(疑似コード):

```python
@bot.on_message
def handle_message(event):
    text = event.message.text.strip()
    if text == "倉庫作業" or "倉庫作業" in text:
        return handle_warehouse_today(event)
    ...
```

### 2. GAS を呼び出す

```python
import requests, os, datetime

def fetch_warehouse_today(date_str=None):
    if date_str is None:
        date_str = datetime.date.today().isoformat()  # 'YYYY-MM-DD'
    resp = requests.post(
        os.environ['EVENT_GAS_URL'],
        json={
            'action': 'warehouse_today',
            'token': os.environ['VEHICLE_RES_TOKEN'],
            'date': date_str
        },
        headers={'Content-Type': 'text/plain'},
        timeout=30
    )
    return resp.json()
```

### 3. レスポンスを整形して返信

```python
def handle_warehouse_today(event):
    data = fetch_warehouse_today()
    if data.get('status') != 'ok':
        reply(event, f"⚠️ 取得エラー: {data.get('message')}")
        return
    date = data['date']
    grouped = data.get('grouped', [])
    if not grouped:
        reply(event, f"今日 ({date}) は倉庫作業の予定がありません。")
        return
    lines = [f"📋 倉庫作業 ({date})"]
    # 会社別にまとめる
    by_company = {}
    for g in grouped:
        co = g.get('company', '')
        by_company.setdefault(co, []).append(g)
    for co, groups in by_company.items():
        lines.append(f"\n▼ {co}")
        for g in groups:
            members = "、".join(g.get('members', []))
            tasks = g.get('tasks', '(タスク未指定)')
            lines.append(f"・{members}: {tasks}")
    reply(event, "\n".join(lines))
```
