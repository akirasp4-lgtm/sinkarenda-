🔑 基本

PIN 8800 で入場。右上の 🔒ロック で一時退出。

画面上部の会社バーで集計対象を切替（全社 or 各社）。

📊 事務タブの使い方

  1. 月別集計: 月を選ぶと職人別の出勤日数・人工合計が表示
  2. CSVダウンロード: 日報・マスタ・集計・操作ログを出力可能
  3. 職人を管理: 氏名・会社・事業部・単価（日当）を登録/編集
  4. 元請を管理: 元請リストの追加/削除
  5. 集計を更新: 全集計シートを一括再生成（毎朝6時も自動）
  6. データアーカイブ: 3ヶ月以上前を退避（毎月1日3時も自動）
  7. 工番を修正する: 事業部変更・売上入力

🔧 工番修正モーダル

現場マスタの全現場が一覧表示されます。各現場で:

  • 🔍 上部の検索ボックスで 元請名・現場名・工番・代表者 を横断検索できます
  • 事業部を変更 → 「工番更新」で新事業部×年度の次連番で再発番（旧工番は欠番）
  • 売上を入力 → 「売上保存」で現場マスタの売上列に反映
  • 代表者の事業部が参考表示されるので、選び直しの判断に使える

🧹 おかしい工番をクリア

事務タブの赤いボタン。休み・倉庫・予定モードや「現場作業以外」のレコードに残っている 工番・事業部 を一括クリアします。

  • 仕様変更以前に登録された「休み なのに工番がついている」レコードを清掃
  • 新規登録時は既に空のまま保存されているため、通常は1回実行すれば十分
  • クリア後は 「集計を更新」 で集計シートを再計算してください

🔀 元請名をマージ

事務タブの青いボタン。同じ会社が違う名前で登録されてしまった場合に、片方を残してもう片方のレコードを統合します。

  • 例: 「グローライズ」と「グローライズ自社」を「グローライズ自社」に統合
  • 統合する元請（消える方）と 残す元請（こちらに統一）を選んで実行
  • 日報・アーカイブ・現場マスタ・元請マスタ 全部を一括書き換え
  • 取り消し不可なので、本当に同じ元請の重複登録か確認してから実行
  • 影響範囲（何件書き換わるか）はプレビューで確認できます

💰 粗利管理の始め方

  1. 職人単価を設定: 職人を管理 → 各職人の単価欄に日当を入力（自動保存）
  2. 現場売上を入力: 工番を修正する → 各現場の売上欄を入力 → 売上保存
  3. 集計ボタン: 事務タブの「集計を更新」

→ スプレッドシートの「事業部別按分」シートで、工番×事業部の人工・原価・売上按分・粗利・粗利率が自動計算されます。

📋 スプレッドシートのシート

  • データ: 日報データ / アーカイブ / 職人マスタ / 元請マスタ / 現場マスタ
  • 集計: 会社別集計 / 月別集計 / 月別確認表 / 元請別請求集計 / 事業部別按分
  • 運用: 操作ログ（誰がいつ何を変更したか自動記録）

⚠ 集計シートは手動で書き込まない（次回集計で消える）

📱 スマホ便利機能

  • 引き下げて更新: 画面を下に引っ張って最新データを取得
  • 📤 共有: 予定詳細から LINE・メール等へ送信
  • 🎤 音声入力: メモ欄でキーボードのマイクから音声入力
  • 📲 ホーム画面追加: アプリのように起動（iPhone は Safari「共有」→「ホーム画面に追加」）
  • 触覚フィードバック: スワイプ・登録時に軽く振動（対応端末のみ）
  • スマホ表示: 空き確認・車確認は自動でリスト形式に切替

🚗 車両セレクタ

予定登録時に、使用する車両をプルダウンから選べます。

  • 「使用しない」がデフォルト。車両不要なら空のままでOK
  • 選んだ日に他の予定で既に予約済みの車両はグレーアウト＋「使用中」表示で選択不可
  • 連続入力（複数日）の場合は、その期間中いずれかの日で予約済みなら選択不可
  • 編集モーダルでも同じ動作（自分自身のグループは除外して判定）
  • 日別モーダルのカードと詳細画面に「🚗 ◯◯」で表示
  • 🏖️休み / 🏭倉庫 をONにすると車両セレクタも自動ロック＆クリア
  • シート上は「車両」列に車種名が保存される（集計シートには影響しない）

登録できる車両: ダブルキャブ / ユニック / 高所作業車 / ハイエース（白・シルバー・白2）/ タウンエース / ProBox / ジムニー / クリッパー（シルバー）

🌙 夜勤モード（職人画面と共通）

  1. 作業日は作業を始める日を選ぶ（翌朝5時に終わる場合は前日）
  2. 🌙夜勤モードをON、出勤・退勤を実時刻で入力（例 22:00 〜 05:00）

退勤＜出勤エラーは夜勤モード時のみ免除。翌日のカレンダー・空き確認に「⏰前日から」表示が自動で出ますが、集計には影響しません。

🆕 休み・倉庫モードの仕様

🏖️休み / 🏭倉庫 トグルON時の挙動

  • 元請・現場・作業区分・出勤/退勤・メモ等の入力欄が自動で無効化される
  • すでに入力されている値は自動でクリアされる
  • 元請名は入力不要（バリデーションをスキップ）
  • 操作者・日付・代表者・同行メンバーは引き続き入力可能
  • もう一度同じトグルを押すと解除され、入力欄が再び有効化される

工番が発行されないモード

  • 🏖️休み / 🏭倉庫 / 📌予定 のいずれかがONの場合は工番が発行されない
  • 🌙夜勤は通常通り工番発行対象

カレンダー表示

  • 元請名なしの休み・倉庫は「🏖️ 休み」「🏭 倉庫」と表示
  • 元請名ありは「🏖️ 〇〇電気」「🏭 〇〇電気」のようにアイコン付き表示

シート集計での扱い

  • 休み: 人工=0、出勤日数にもカウントされない
  • 倉庫: 人工=1.0、出勤日数に+1（通常勤務と同じカウント）
  • 倉庫レコードに元請名がない場合、元請別請求集計・事業部別按分には載らない（工番なし・元請なしのため）

📊 1日複数現場の按分ルール（A案）

1人が同じ日に複数現場へ行ったら、自動で 1人工を 1/N に按分。

  • 昼A + 昼B → 各 0.5（合計 1.0）
  • 3現場 → 各 1/3
  • 昼A + 夜B → A=1.0、B=1.0（昼夜は別カウント）

影響シート: 元請別請求集計 / 事業部別按分。月別確認表は1日1人工で頭打ち。

🏷 作業区分

現場名は統一したまま「現調 / 着打ち / カギ借用 / 材料引き取り / 撤去品返却 / その他」で分類可能。

集計上は人工合計に合算。詳細画面・月別確認表では区分が見えるよう表示。

📅 カレンダー / 詳細画面 / スワイプ

  • 日付タップ → 日別モーダル（現場名+メンバー+モードアイコンの最小表示）
  • 予定タップ → 詳細フルスクリーン（編集・削除・コピー集約）
  • メモ内URLは自動でタップ可能リンクに変換
  • 横スワイプ: 月切替/前後日/前後週/詳細前後

🏗️ 現場管理タブ

  • 元請名 + 月で絞り込み可能（月は元請の右側、デフォルトは「全期間」）
  • 月を選ぶとその月の現場リスト・日程一覧だけが表示される
  • チェックボックスで複数日選択 → 一括編集で時間・元請・代表者・メンバー・車両等をまとめて変更
  • 編集保存時、メンバーが他の予定とぶつかる場合は被り警告が出ます

🆘 困ったら

  • 工番が空欄 → 代表者の事業部未設定。職人管理で割当
  • 粗利がマイナス → 売上未入力 or 単価が高すぎる可能性
  • 原価が0 → 職人の単価が未設定
  • 起動が遅い → アーカイブを実行（3ヶ月以前を退避）
  • 事業部別按分で「不明」 → 職人マスタで事業部を割当 or 工番修正で現場の事業部確定
  • 現場マスタに使ってない現場名が残る → 集計実行時に自動掃除（売上未入力＆日報参照なし）
工番事業部の修正

各現場の事業部を選び直して「更新」で反映。新事業部×年度の次連番で新工番を発番し、現場マスタ＋日報データ＋アーカイブを一括更新します。旧工番は欠番として残ります。

🔀 元請名をマージ

「統合する元請」のレコードを「残す元請」の名前に書き換えます。日報・アーカイブ・現場マスタ・元請マスタ全部一括で書き換えます。この操作は取り消せません。
