🔑 基本

PIN 8800 で入場。右上の 🔒ロック で一時退出。

画面上部の会社バーで集計対象を切替（全社 or 各社）。

📊 事務タブの使い方

  1. 月別集計: 月を選ぶと職人別の出勤日数・人工合計が表示
  2. CSVダウンロード: 日報・マスタ・集計・操作ログを出力可能
  3. 職人を管理: 氏名・会社・事業部・単価（日当）を登録/編集
  4. 元請を管理: 元請リストの追加/削除
  5. 集計を更新: 全集計シートを一括再生成（毎朝6時も自動）
  6. データアーカイブ: 3ヶ月以上前を退避（毎月1日3時も自動）
  7. 工番を修正する: 事業部変更・売上入力

🔧 工番修正モーダル

現場マスタの全現場が一覧表示されます。各現場で:

  • 事業部を変更 → 「工番更新」で新事業部×年度の次連番で再発番（旧工番は欠番）
  • 売上を入力 → 「売上保存」で現場マスタの売上列に反映
  • 代表者の事業部が参考表示されるので、選び直しの判断に使える

💰 粗利管理の始め方

  1. 職人単価を設定: 職人を管理 → 各職人の単価欄に日当を入力（自動保存）
  2. 現場売上を入力: 工番を修正する → 各現場の売上欄を入力 → 売上保存
  3. 集計ボタン: 事務タブの「集計を更新」

→ スプレッドシートの「事業部別按分」シートで、工番×事業部の人工・原価・売上按分・粗利・粗利率が自動計算されます。

📋 スプレッドシートのシート

  • データ: 日報データ / アーカイブ / 職人マスタ / 元請マスタ / 現場マスタ
  • 集計: 会社別集計 / 月別集計 / 月別確認表 / 元請別請求集計 / 事業部別按分
  • 運用: 操作ログ（誰がいつ何を変更したか自動記録）

⚠ 集計シートは手動で書き込まない（次回集計で消える）

🆘 困ったら

  • 工番が空欄 → 代表者の事業部未設定。職人管理で割当
  • 粗利がマイナス → 売上未入力 or 単価が高すぎる可能性
  • 原価が0 → 職人の単価が未設定
  • 起動が遅い → アーカイブを実行（3ヶ月以前を退避）
工番事業部の修正

各現場の事業部を選び直して「更新」で反映。新事業部×年度の次連番で新工番を発番し、現場マスタ＋日報データ＋アーカイブを一括更新します。旧工番は欠番として残ります。
