## ★改訂（重要）：1つのひな型 → 各局ファイルを土台に

実データ5局を読んで、**固定値が局ごとに違う**と判明したため方式を変えた：
- **法定保険料率が局で違う**（F95系＝16.36% / 4G系（アムール等）＝15.765%）
- **工番の頭が違う**（F95 / F96）。工事名・施工場所・注文金額・労務費も各局固有
- → 1つのひな型(16.36%・F95)で全局を作ると料率・工番を間違える

**新方式**：各局は自分の実ファイル（固定値が正しく入っている）を**土台**にし、毎月変わるセル
（請求書 `AO17`当月迄/`AO18`前月迄/`CO1`請求日、表紙 `AA5`請求年/`AE5`請求月）**だけ**を書き換える。
固定情報（料率`H15`・工番`J54/M54/P54`・注文金額`G8`・労務費`H14`・工事名`G28`…）は**一切触らない**。
局マスタは局ごとに `{file, 注文金額(残額表示用), 前月迄出来高}` だけ持つ（固定情報はファイル側が正）。
下記「穴埋めするセル」の表紙側の大半（金額/労務費/工事名/工期/工番…）は**旧方式の記録**で、新方式では触らない。

## Context（なぜやるか）

きんでん西（株式会社きんでん 情報通信工事センター）は、請求を**きんでん指定の様式**で、しかも**1局ずつ**出す必要がある。他の元請は「現場ごと請求」（前日実装＝`請求計算`タブ）のままでよい。指定様式は体裁が厳格で、崩すと受理されない（同梱の「見積提出方法」シートに明記）。

サンプル実物：`_local/kinden_seikyu.xlsx`（経理の取引先別フォルダにある「指定見積書表紙・工事代金請求明細書.xlsx」のコピー。元パスは社内NAS）。

## きんでん書式の構造（実ファイルより）

ワークブックは4シート。生成で扱うのは先頭2つ：

1. **「きんでん情通C指定見積表紙」** … 指定見積書の表紙。工事代金（注文金額）＋**法定福利費（＝労務費 × 法定保険料率16.36%、セル数式 `=H14*H15`）**＋工事名/工番/施工場所/工期＋（固定の）建設業許可・協力会社コード・会社情報。
2. **「請求書(インボイス）」** … 工事代金請求明細書。**出来高請求**：注文金額に対し `当月出来高(C)=当月迄(A)−前月迄(B)`、`消費税(D)=C×税率`、`請求合計(E)=C+D`。(A)(B)以外は数式。適格請求書番号は固定値。
3. 見積記入例 / 見積提出方法（参照のみ・生成しない）

> 請求方式：**出来高分割**。当月の金額は**きんでん指示**で来る（こちらで進捗計算しない）。前月迄(B)はこちらで累計を覚える。

## 設計

### 方式：ひな型穴埋め（Approach A）
ゼロから再構築せず、**実ファイルをひな型にして「局で変わるセルだけ」上書き**する。会社情報・インボイス番号・建設業許可・協力会社コード・保険料率は**全局共通の固定値**なのでひな型に入ったまま触らない。これで体裁・数式・印影を100%維持する。

実装は **Python で xlsx（zip）内のシートXMLのセル値だけを直接書き換え**、画像・印影・スタイル・結合・数式は一切触らない（＝バイト単位で温存）。
- **openpyxl は使わない**：このひな型には印影/ロゴ画像が4点（PNG×2・JPEG・EMF）入っており、openpyxl で読み書きすると EMF が落ち他画像もズレるため、指定様式が崩れる。
- 対象セルは全て XML 内に実在（空セルも自己終了タグで存在）。数値は `<v>` 差し替え、文字列は inlineString 化。**style属性(`s=`)は読み取って維持**。
- 値変更後に数式が再計算されるよう `workbook.xml` の `calcPr` に `fullCalcOnLoad="1"` を立てる（法定福利費・当月出来高・消費税・合計が開いた瞬間に正しく出る）。
- PWA/GAS は Excel→Googleスプレッドシート変換で体裁が崩れるため使わない。

### データの持ち方
- **局マスタ**（局ごとに最初に1回設定、以後使い回し）：
  注文金額 / 労務費 / 工事名 / 工番 / 施工場所 / 工期(着工〜完成) / 見積No / **前月迄出来高(累計)**
- **共通定数**（全局同じ・ひな型に保持、リポジトリには出さない）：
  グローライズ会社情報・住所・電話 / 適格請求書番号 / 建設業許可情報 / 協力会社コード / 法定保険料率16.36% / 税率10%
- **月次入力**：局ごとの「当月迄出来高(A)」（きんでん指示）。前月迄(B)はマスタから自動。

### 穴埋めするセル（局で変わる所だけ）

**シート「きんでん情通C指定見積表紙」**

| セル | 内容 |
|---|---|
| `AB4` | 見積No |
| `AA5`/`AE5`/`AH5` | 見積日（西暦/月/日） |
| `G8` | 工事代金（注文金額・取決金額） |
| `H14` | 労務費（→`H16=H14*H15`で法定福利費が自動） |
| `G28` | 工事名 |
| `G30` | 施工場所(住所) |
| 着工 `G32,H32,I32,J32`(年) `L32,M32`(月) `O32,P32`(日) | 着工年月日（1桁ずつ） |
| 完成 `W32,X32,Y32,Z32`(年) `AB32,AC32`(月) `AE32,AF32`(日) | 完成年月日（1桁ずつ） |
| `M54`,`P54` | 工番の可変部（`Ｆ95-{M54}-{P54}`。`I54=Ｆ`/`J54=95` は固定） |

**シート「請求書(インボイス）」**

| セル | 内容 |
|---|---|
| `AO17` | 当月迄出来高(A)（きんでん指示の累計） |
| `AO18` | 前月迄出来高(B)（マスタの累計＝前回の(A)） |
| `CO1` | 請求日（月末日。年月は表紙参照） |

> `AO16`(=表紙!G8)・`AO19`(=A−B)・`AO20`(=消費税)・`AO21`(=合計) は数式なので触らない。

### 月次フロー
1. きんでんから各局の「当月迄出来高」指示を受け取る
2. 局名＋当月迄出来高を渡す
3. 生成：ひな型コピー→局マスタ＋当月迄出来高で該当セルを上書き→**局ごとに完成xlsxを出力**（ファイル名例：`きんでん_{局名}_{年月}.xlsx`）
4. 生成後、マスタの「前月迄出来高」を今回の「当月迄出来高」に更新（次回の(B)になる）

### 当面の運用
生成はPythonスクリプト（openpyxl）。**当面は依頼ベース**＝利用者は「局名と当月迄出来高」を伝えるだけ、生成は私が実行して完成ファイルを渡す。本数が増えたら入力フォーム化や自動化を検討。

### 配置とセキュリティ（重要）
- ひな型 `_local/きんでん/template.xlsx`、局マスタ `_local/きんでん/局マスタ.*`、出力 `_local/きんでん/out/` … **すべて `_local/`（gitignore済・公開リポに上げない）**。きんでんの指定様式・取引金額・許可番号・インボイス番号等の機密を公開先に出さないため。
- 生成スクリプト（コードのみ・機密を含まない）はリポジトリに置いてよい。

## やらないこと（この段階の範囲外）
- PWA/GAS からのワンクリック生成（体裁リスクのため見送り）
- きんでん以外の元請（既存の「現場ごと請求」のまま）
- 見積記入例・提出方法シートの生成（参照のみ）
- 出来高％の自動算出（金額はきんでん指示）

## 検証
1. ひな型をコピーして1局穴埋め→Excelで開き、法定福利費(`H16`)・当月出来高(`AO19`)・消費税(`AO20`)・合計(`AO21`)の数式が正しく再計算される
2. 体裁（罫線・結合・印影・レイアウト）がひな型と同一（差し替えセル以外を触っていない）
3. 前月迄→当月迄の累計繰越が、翌月生成で(B)に正しく入る
4. 1局＝1ファイルで出力される
