# 祝日カレンダー表示 設計書

最終更新: 2026-06-15
対象プロジェクト: 予定管理アプリ作成（sinkarenda-）

## 目的

カレンダー上に日本の祝日を表示する。職人・社長・管理者の3画面すべてで、祝日の
日付を赤くし、祝日名（例「海の日」）を小さく添える。既存の予定表示・土日色分け・
車両表示などには一切手を入れず、独立した追加として実装する。

## 決定事項（利用者承認済み）

- **見せ方**: 日付番号を赤（日曜と同じ `#E8384F`）＋その下に小さく祝日名
- **対象画面**: `index.html`（職人）/ `president.html`（社長）/ `admin.html`（管理者）
- **データ方式**: A＋C の二段構え（下記）。**GASは触らない＝HTML修正のみ**

## データ方式：自動取得＋端末記憶＋内蔵フォールバック

1. **取得元**: `https://holidays-jp.github.io/api/v1/date.json`
   （`{"YYYY-MM-DD":"祝日名", ...}` を返す公開JSON。CORS許可・無料）
2. **キャッシュ**: 取得成功時に `localStorage` へ `{fetchedAt, data}` で保存。
   次回以降、キャッシュが新しければ（目安30日以内）それを使い、古ければ再取得。
3. **フォールバック（内蔵）**: 通信できず・キャッシュも無い場合に備え、
   当年＋翌年ぶんの祝日を各HTMLに直接埋め込む（初回オフラインでも表示される）。
4. 取得は非同期。読み込み完了後に `renderCalendar()` を呼び直して反映する
   （初期描画はフォールバック/キャッシュで即表示 → 取得完了で差し替え）。

### 優先順位（どれを表示に使うか）
新しいキャッシュ > 取得成功データ > 古いキャッシュ > 内蔵フォールバック

## 実装ポイント（画面別）

共通の祝日オブジェクト（例 `JPH`：`{ '2026-07-20': '海の日', ... }`）を各HTMLに持ち、
`renderCalendar` 内で「その日が `JPH` に在れば赤クラス＋名前」を足す。

- **index.html / admin.html**（HTML文字列方式・`cal-cell`/`cal-day-num`/`sun`/`sat`）
  - 祝日セルに `holiday` クラスを付与
  - `cal-day-num` の直後に `<div class="cal-holiday-name">祝日名</div>` を挿入
  - CSS追加: `.cal-cell.holiday .cal-day-num{color:#E8384F}` /
    `.cal-holiday-name{font-size:9px;color:#E8384F;text-align:center;line-height:1.1;overflow:hidden;text-overflow:ellipsis;white-space:nowrap}`
- **president.html**（DOM生成方式・`cal-day`/`day-num`/`sun`/`sat`・62pxの小型セル）
  - `day-num` 要素に祝日なら `holiday` クラス（赤）
  - 小さな祝日名要素を追加。マスが小さいため極小フォント＋1行省略
  - 当月外セル（`other`）も祝日色は付けてよい（既存の薄表示はそのまま）

### 既存挙動との関係
- `today`（緑丸＋白文字）が最優先で見た目が決まる（祝日が今日でも緑丸が勝つ＝問題なし）
- 祝日が日曜/土曜と重なる場合も、番号は赤（日曜赤と同色なので違和感なし。土曜青より祝日赤を優先）
- 予定チップの表示・件数・並びは不変

## 非対象（YAGNI）

- 祝日を「休み予定」として自動登録する等の業務ロジックは作らない（表示のみ）
- 振替休日・国民の休日の独自計算はしない（取得元JSONが既に含むため不要）
- GAS・スプレッドシート・車両・請求などのバックエンドは一切変更しない

## テスト/確認

- 既知の祝日日付（例 2026-07-20 海の日、2026-01-01 元日）が赤＋名前で出る
- 平日の祝日で番号が赤くなる／祝日名が出る
- 通信を切っても内蔵フォールバックで当年祝日が出る
- 3画面とも既存の予定・土日色・今日マークが従来どおり
- 各HTMLのインラインJS構文エラー0（node 構文チェック）

## デプロイ

HTML 3ファイルのみ変更 → GitHub へ push → GitHub Pages 自動反映。
**GAS の再デプロイは不要**。
