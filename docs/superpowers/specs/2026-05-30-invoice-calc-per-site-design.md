# 設計書：請求計算タブを「現場ごと」に作り直す（第1段階・改）

最終更新：2026-05-30
ステータス：設計承認済み（口頭）・実装プラン未作成
前提設計：`docs/superpowers/specs/2026-05-29-invoice-calculation-design.md`（第1段階の初版）

## なぜ作り直すか（舟田さんの言葉に立ち返る）

舟田さん（グローライズ・事務）の請求パターンは**3つ**ある（初版設計書 Context より）：

| パターン | 中身 | このアプリでの扱い |
|---|---|---|
| **①部分検収** | 元請の書式に記入。総請負金額を部分（着手/中間/完工…）に分けて請求 | **次段階**（元請書式の請求書を自動生成）。サンプル＝`【請求書】深谷市系統用蓄電池設置工事1.xlsx` |
| **②常用・日当** | 出面数 × 日当 ＋ 経費 | **本書の対象**。現場ごとに作り直す |
| **③請負** | 進捗・残額管理 | 次段階 |

第1段階（初版）で②を実装したが、**粒度が「現場×職人」になっていた**。舟田さんの実務では「職人ごとに単価は変わらない／応援は単価×人数でいい」ため、**現場ごと**に集約するのが正しい。本書はその修正＋請負現場の最小対応を定義する。

> サンプル請求書は舟田さんの言う**①部分検収**（けんしゅう。音声では「部分研修」と聞こえた）。総請負金額15,000,000円を着手30%/中間30%/完工40%で部分請求している。**これ自体の自動生成は本段階では作らない**（次段階）。

## 本段階のゴール

admin の「💰請求計算」タブで、元請＋月を選ぶと**現場ごと**に1行で請求額が出る：
- **応援（常用・日当）の現場** … 延べ出面 × その現場の単価 ＋ 経費
- **請負の現場** … 今回の請求金額を手入力（出面・単価計算はしない）

関数入りExcelで現場ごとの合計を出力できる（既存方式を踏襲）。

## 変更点（初版との差分）

### 1. 単価マスタを「元請×現場」キーに（職人を外す）
- 「請求単価マスタ」シート：`元請名 / 現場名 / 職人名 / 単価 / 更新日時`（5列）→ **`元請名 / 現場名 / 単価 / 更新日時`（4列）**
- キー＝**元請 × 現場**。職人は持たない（単価は職人で変わらないため）
- 本番の単価データは現状ゼロ（`get_billing_rates`→`rates:[]` 確認済み）なので移行リスクなし。`getOrCreateBillingRateSheet_` でヘッダが旧形式なら新形式へ寄せる

### 2. 請求計算タブを現場ごとに集約
- 元請＋月で絞った出面を、**現場（loc）単位**で集計（職人をまたいで合算）
- 延べ出面 ＝ その月にその現場へ出た**のべ人日**（3人×4日＝12）。倉庫/休み/予定（souko/yasumi/yotei）は除外（既存フィルタ踏襲）
- 各行は方式で分岐：

| 方式 | 現場 | 延べ出面 | 単価 | 金額 | 経費 |
|---|---|---|---|---|---|
| **応援** | 自動 | 自動 | 元請×現場で記憶／入力 | =出面×単価（自動） | 手入力（任意） |
| **請負** | 自動 | —（無効） | —（無効） | **手入力**（今回請求額） | —（無効） |

- 方式は現場マスタ「請求方式」列（既存・応援/請負の2値）を流用。行で切替→現場単位で保存（既存 `update_site_billing_method`）
- **請負の「請求方式」は、舟田さんの①部分検収・③請負をまとめて「金額手入力」で扱う簡易対応**。①部分検収の書式生成・③進捗残額は次段階で細分化する
- 単価は入力すると元請×現場で記憶（次回自動表示）
- 請負の手入力金額は**今回の出力用**（毎回額が変わるため記憶しない。部分払いの記憶は次段階）

### 3. Excel出力（現場ごとの行）
- 「請求計算」シートを現場ごとの行で生成：
  - 応援行：金額セル ＝ `出面セル × 単価セル`（関数）
  - 請負行：金額セル ＝ 手入力した数値（関数なし）
  - 経費列（応援のみ）
  - 合計 ＝ `SUM(金額)` ＋ `SUM(経費)`
- 既存 `generate_billing_calc_xlsx` を現場ごとの payload に合わせて改修（`exportSheetAsXlsxBase64_` 踏襲）

## GAS 変更（gas.js）

| アクション | 変更 |
|---|---|
| `get_billing_rates` | 戻りを `{genba, loc, rate}`（職人を外す） |
| `save_billing_rate` | キーを元請×現場に（upsert） |
| `generate_billing_calc_xlsx` | payload を現場ごと `{loc, method, manDays, rate, amount, expense}` に。応援＝`=C*D`、請負＝amount数値 |
| `update_site_billing_method` | 変更なし |
| `getOrCreateBillingRateSheet_` | 4列ヘッダに（旧5列なら寄せる） |

## admin.html 変更

- `billingRate(genba, loc)`：職人キーを外す
- `getBillingLines()`：現場（loc）単位に集約（職人の出面を合算）、方式を付与。請負は手入力金額を持つ
- `renderBillingTable()`：現場ごとの行。応援＝単価/金額/経費、請負＝金額手入力（他は無効）
- `onBillingRateInput()`：元請×現場キー（行ごと金額更新は維持）
- `exportBillingXlsx()`：現場ごと payload。単価保存は元請×現場。請負は手入力額を載せる
- `updateBillingTotal()`：金額（応援＝計算／請負＝手入力）＋経費の合計

## 影響範囲
- admin.html のみ（index.html＝職人用は変更なし）
- 単価マスタは本番空なので移行リスクなし
- カレンダー・集計・請求集計シートには手を入れない
- GAS 再デプロイ必要（**v37 予定**、同じデプロイID `AKfycbxp2eUc…` を更新）

## 検証
1. 元請＋月 → **現場ごと**に1行（職人ごとに並ばない）
2. 応援現場：延べ出面 × 単価 ＝ 金額、経費を足して合計
3. 単価入力 → 再読込で自動表示（元請×現場で記憶）
4. 方式「請負」に切替 → 出面/単価/経費が無効、金額は手入力できる
5. 請負の手入力金額が合計に入る
6. Excel出力 → 応援行は出面/単価を変えると金額・合計が自動再計算、請負行は手入力額が入る
7. 「請求単価マスタ」が4列（職人なし）になっている

## 既知の制約
- 行はその月に**出面のある現場**から作る。請負現場でその月に出面記録が無い場合（例：完工分を出面ゼロの月に請求）は行に出ない。通常は現場へ人が出ているので出るが、出ないケースの手動追加は本段階では作らない（次段階で検討）。

## やらないこと（本段階の範囲外）
- ①部分検収＝元請書式の請求書そのものの自動生成（サンプルxlsxの形）→ **次段階で専用設計**
- ③請負の進捗・残額管理 → 次段階
- 職人ごとの単価・明細

## 次段階への申し送り
- **①部分検収**：元請ごとに書式が違う（サンプル＝ハイテックス書式：総請負金額＋着手/中間/完工の割合＋消費税＋振込先）。総額×割合で部分請求し、合計が総額に戻る。書式テンプレ方式を要設計
- **③請負**：契約金額に対する進捗％・残額
