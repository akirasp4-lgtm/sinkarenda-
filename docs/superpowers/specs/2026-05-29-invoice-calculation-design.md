# 設計書：請求計算（常用・日当）の自動化

最終更新：2026-05-29
ステータス：設計承認済み・実装プラン未作成

## Context（なぜやるか）

舟田さん（グローライズ・事務）の毎月の請求書作成が手作業で非常に煩雑。音声ヒアリングで判明した課題：

- 出面表をチェックして「今月どの元請の仕事をしたか」を特定 → 自社Excelに転記 → 請求パターン別に金額計算 → 送付、という流れを全部手作業。
- 請求パターンが3つあり、それぞれ計算が違う：①部分検収（元請書式に記入）②**常用・日当（出面数×日当＋経費）**③請負（進捗・残額計算）。
- 特に「過去の履歴を遡って単価や前月分を確認する」のが面倒。

予定管理アプリには**出面データ（誰が・いつ・どの元請の現場に出たか）は既にある**が、そこから先（金額計算・経費・帳票）が全部手作業。出面データを活かせば請求計算を自動化できる。

## スコープ（段階分け）

請求書業務全体は大きいので3段階に分割し、**本設計書は第1段階のみ**を対象とする：

| 段階 | 内容 | 本書の対象 |
|---|---|---|
| **第1段階** | **②常用・日当の請求額を自動算出**（出面×単価＋経費、金額が出るまで） | ✅ 本書 |
| 第2段階 | 請求書そのもの（PDF/Excel帳票）の自動生成 | 次回 |
| 第3段階 | ③請負案件の進捗・残額管理 | 次回 |

第1段階のゴール：**「今月この元請にいくら請求するか」が画面で自動算出され、関数入りExcelで出力できる**。請求書の体裁づくりは第2段階。

## 第1段階の設計

### 全体像

admin.html（事務用）に **「💰 請求計算」タブ**を新設。元請＋月を選ぶと、出面データ（既存）から明細が自動で並び、単価を入れれば金額が出る。「Excel出力」ボタンで関数入り xlsx を生成。

### データモデル

**(A) 新シート「請求単価マスタ」**（BILLING_RATE_SHEET）
- 列：`元請名 / 現場名 / 職人名 / 単価 / 更新日時`
- キー＝**元請 × 現場 × 職人**の3点セット
- 請求計算画面で単価を入力すると upsert（同キーがあれば更新、なければ追加）
- 次回以降、同じ組み合わせは単価を自動表示 → **「履歴を遡る」が消える**
- 「単価は基本ずっと同じ」というヒアリング結果に基づく設計

**(B) 現場マスタ（JOBSITE_SHEET）に「請求方式」列を追加**
- 現状9列（元請名/現場名/工番/事業部/年度/連番/売上/読み/完了）→ **10列目「請求方式」**を追加（自動マイグレーション）
- 値：`応援`（＝常用・日当）/ `請負`。空欄は「応援」扱い（デフォルト）
- 「応援/請負はだいたい現場で決まる」を反映した初期値。請求計算画面で行ごとに上書き可。

### 請求計算タブの動き

1. **元請 + 月**を選ぶ
2. その元請のその月の出面を、**現場 × 職人**で集計して明細行を表示（出面数は既存データから自動）
3. 各行の構成：

| 列 | 内容 | 入力/自動 |
|---|---|---|
| 現場 | 現場名 | 自動 |
| 職人 | 職人名 | 自動 |
| 出面数 | その月の出面合計（人日） | 自動（既存データ） |
| 方式 | 応援 / 請負 | 現場マスタ初期値→行ごと切替可 |
| 単価 | 日当単価 | 請求単価マスタから自動表示／初回は入力 |
| 金額 | 出面数 × 単価（**応援のみ**） | 自動計算 |
| 経費 | ETC・ガソリン等（**応援のみ**入力欄） | 手入力（任意） |

4. 単価を入力/変更すると **請求単価マスタに保存**（次回自動表示）
5. **請負の行は「出面×単価」計算の対象外・経費欄も無効**（請負の中身は第3段階）
6. 元請ごとの合計（金額＋経費）を表示

### 出面集計のデータ源

既存のフロント関数を流用：`filteredNippos()`（権限フィルタ済み全予定）→ 元請・月で絞り、現場×職人ごとに出面数を数える。休み・予定・倉庫の扱いは既存の集計ロジック（`yakin !== '休み' && yakin !== '予定'`）に合わせる（倉庫を含めるかは実装時に確認）。

### Excel出力（関数入り）

- 「Excel出力」ボタンで xlsx を生成・ダウンロード
- **金額セル = `出面数セル × 単価セル`、合計 = `SUM(...)`** を Excel 関数として埋め込む
- → Excel側で出面数や単価を直すと金額・合計が自動再計算される
- 実装は既存の **shared formula 方式**（月別確認表 xlsx で実績あり：`exportSheetAsXlsxBase64_` + `<f t="shared">` パターン、引き継ぎ commit `85b84c9`）を踏襲

### GAS 新規アクション（gas.js）

| アクション | 役割 |
|---|---|
| `get_billing_rates` | 請求単価マスタを全件返す（フロントで組み合わせ→単価を引く） |
| `save_billing_rate` | 単価を upsert（元請×現場×職人→単価） |
| `update_site_billing_method` | 現場マスタの「請求方式」を更新 |
| `export_billing_xlsx` | 請求計算の明細を関数入り xlsx で出力 |

`doGet` 応答に現場マスタの「請求方式」を含める（既存の `jobsites` 配列に `billingMethod` を追加）。

## 影響範囲

- **admin.html のみ**にタブ追加（index.html＝職人用は変更なし）。
- 現場マスタへの列追加は自動マイグレーションで既存データ非破壊。
- 既存の集計・請求集計シート・カレンダー機能には手を入れない（独立機能として追加）。
- GAS 再デプロイが必要。

## 検証（Verification）

1. admin →「💰 請求計算」タブ → 元請＋月を選ぶ → 現場×職人×出面数が自動で並ぶ
2. 単価を入力 → 金額（出面×単価）が即計算される
3. 別の月に切り替え→戻る → 入れた単価が**自動表示**される（マスタ保存の確認）
4. 行の方式を「請負」に切替 → その行は金額・経費が対象外になる
5. 経費を入力 → 合計に反映（応援の行のみ）
6. 「Excel出力」→ xlsx をダウンロード → Excelで開き、**出面数や単価を変えると金額・合計が自動再計算**される（関数が入っている）
7. 現場マスタに「請求方式」列が増え、既存データが壊れていないこと
