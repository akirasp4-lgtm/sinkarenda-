# 設計書：現場管理タブの「探す」強化 ＋ 倉庫を請求集計に追加

最終更新：2026-05-29

## Context（なぜやるか）

利用者（建設会社の事務）から「カレンダー 修正希望」として4点の要望。①現場別管理ページは実装済みなので、残る3点を対応する：

- **②個人名でも探したい** — 今は「現場名」でしか絞れない。「○○さん最近どこ行ってた？」を追えない。
- **③予定（📌ピン）の一覧を出したい** — 月末に「📌のまま放置された予定」を不特定多数の現場から探すのが大変。
- **④倉庫作業も「元請別請求集計_フィルタ用」シートに出したい** — 人別で「現場作業か倉庫作業か」によって日当が変わることがあるため、倉庫の出面も人別×日別で把握したい。今は集計から除外されている。

ゴール：現場管理タブを「現場／人／予定ピン」で切り替えて探せる画面にし、倉庫をフィルタ用シートに載せて日当計算に使えるようにする。

---

## 全体方針

3機能とも **既存の部品を流用** して最小改修で実現する。新しいタブは作らない（スマホでタブが窮屈になるため）。

- ②③：現場管理タブ（`screen-genba`）の上部に「探し方の切り替えボタン」を足し、下の日程一覧・編集・削除UIはそのまま使い回す。
- ④：GAS の `generateBillingFilterSheet_` だけを直す。通常の請求集計シートは触らない。

---

## ② ③ フロントエンド設計（index.html / admin.html 両方）

### 現状の部品（流用元）

| 関数 | 役割 | index.html |
|---|---|---|
| `filteredNippos()` | 権限フィルタ済みの全予定 | 既存 |
| `groupNippos(nippos)` | 予定を1日1グループにまとめる | 既存 |
| `getGmGroups()` | 現場名で絞ったグループ配列を返す | L2611 |
| `renderGmList()` | グループ配列を一覧描画＋編集/削除ボタン | L2623 |
| `openEditModal(idx, groups)` | 行タップで詳細→編集 | 既存 |

### 改修のキモ：データ源を「モード別」に差し替える

現状 `renderGmList()` と編集ボタンは `getGmGroups()` を直に呼んでいる。これを **現在のモードに応じてグループを返す共通関数** に置き換える：

```js
let gmMode = 'genba'; // 'genba' | 'person' | 'pin'

function getActiveGmGroups(){
  if(gmMode==='person') return getPersonGroups();
  if(gmMode==='pin')    return getPinGroups();
  return getGmGroups(); // 既存の現場別
}
```

- `renderGmList()` 内の `getGmGroups()` → `getActiveGmGroups()` に変更。
- 各行の編集ボタン `openEditModal(${idx}, getGmGroups())` → `openEditModal(${idx}, getActiveGmGroups())`。
- 一括編集 `openBulkEditModal()` / 削除 `deleteGmChecked()` 内の `getGmGroups()` も `getActiveGmGroups()` に統一。

### 切り替えUI（HTML）

`screen-genba` 上部、現状の「元請名/月」カードの**前**にモード切り替えボタンを追加：

```
[🏗️ 現場で探す]　[👤 人で探す]　[📌 予定だけ]
```

押すと `setGmMode('genba'|'person'|'pin')` を呼び、3種類のフィルタUIカードを出し分ける（同じ場所で表示/非表示を切り替え）：

- 現場モード：今の「元請名/月 → 現場名」カード（そのまま）
- 人モード：「人を選ぶ（職人マスタ）＋月（任意）」のカード（新規）
- 予定ピンモード：「期限切れのみ／月で絞る」の切り替え（新規）

### ② 人で探す：`getPersonGroups()`

- 選択した人が **代表 or メンバーのどちらか**で含まれる予定を抽出。
  `filteredNippos()` を `groupNippos()` でまとめた後、各グループの `members` に選択者名が含まれるものだけ残す。
- 月が選ばれていれば日付で絞る。
- 日付順ソート。
- 一覧の各行は既存表示（日付・メンバー・バッジ）でOK。役割（代表/同行）は `members` から拾えるので、その人の役割バッジを1つ足してもよい（任意）。

### ③ 予定だけ：`getPinGroups()`

- `filteredNippos()` から `yotei`（📌予定）フラグが立った予定だけ抽出 → `groupNippos()`。
- **デフォルト（期限切れモード）**：`date < 今日` のものだけ。月末の「片付け忘れ」を炙り出す用。赤系で目立たせる。
- **月で絞るモード**：選んだ月の📌を全部。
- 行タップ → 既存の詳細/編集（作業区分を「現場作業」等に変えれば📌が外れて確定になる）/削除。

### 影響範囲

- カレンダー・空き確認・車確認・事務タブには影響しない（`screen-genba` 内に閉じる）。
- `openEditModal` / `openBulkEditModal` / `deleteGmChecked` は groups を引数で受け取る設計なので、データ源差し替えだけで全モードで動く。
- admin.html も同じ構造なので同じ改修を入れる。

## ④ 倉庫を「元請別請求集計_フィルタ用」に出す（gas.js）

### 対象関数

`generateBillingFilterSheet_(ss, records)` — gas.js L1838。**この関数だけ**を直す。
通常の `generateBillingSummary_`（L1733、元請への請求書そのもの）は **倉庫除外のまま**にする。

### 現状の除外（L1847）

```js
const workRecords = records.filter(r => r.yakin !== '休み' && r.yakin !== '予定' && r.yakin !== '倉庫');
```

倉庫（`yakin==='倉庫'`）が全部落ちている。

### 改修方針

1. 倉庫を含める（休み・予定は引き続き除外）。倉庫行は **現場名を「倉庫作業」に正規化** し、元請名（genba）は空のままにする：

```js
const workRecords = records
  .filter(r => r.yakin !== '休み' && r.yakin !== '予定')
  .map(r => r.yakin === '倉庫'
    ? Object.assign({}, r, { genba: '', loc: '倉庫作業' })
    : r);
```

2. 集計ループ（L1877 `genbas.forEach`）は元請名でブロックを作るが、倉庫は genba が空。現状 `genbas` は `filter(Boolean)` で空を除外しているため、倉庫が拾われない。
   → **倉庫がある場合は genbas に空文字ブロックを1つ加える**。空 genba ブロック内では loc が「倉庫作業」で人別×日別が出る。

```js
const genbas = [...new Set(workRecords.map(r => r.genba).filter(Boolean))].sort();
if (workRecords.some(r => r.loc === '倉庫作業' && !r.genba)) genbas.push(''); // 倉庫ブロック
```

3. 出力イメージ（フィルタ用シート、ヘッダー＝月／会社名(=元請名)／現場名／名前／1〜31日／合計）：

| 月 | 会社名 | 現場名 | 名前 | … 日別 … | 合計 |
|---|---|---|---|---|---|
| 2026年5月 | （空） | 倉庫作業 | 田中 | 1,1,… | 5 |
| 2026年5月 | （空） | 倉庫作業 | 合計 | … | 8 |

→ **フィルタで「現場名＝倉庫作業」**を選べば「誰が何日倉庫やったか」が出る＝日当計算に使える。
→ **元請名（会社名列）で絞れば**倉庫（空欄）は外れるので、元請への請求一覧には混ざらない。棲み分け成立。

### 影響範囲

- フィルタ用シートのみ。通常の請求集計シート・月別集計・事業部按分には影響しない。
- 按分ロジック（`sitesByPDN`）：同日に現場＋倉庫を掛け持ちした人は 0.5＋0.5 と按分される（論理的に正しい挙動。稀ケース）。
- GAS 再デプロイが必要（フロントは GitHub Pages、GAS は別途デプロイ）。

## 検証（Verification）

### ② 人で探す
1. 現場管理タブ →「👤 人で探す」→ 人を選ぶ → その人が代表/メンバーで入った予定が日付順に出る。
2. 月で絞ると当該月だけに。
3. 行をタップ → 詳細→編集/削除が従来通り動く。

### ③ 予定だけ
4. 「📌 予定だけ」→ 初期表示で「今日より前の📌」だけが赤系で並ぶ。
5. 「月で絞る」に切り替え → その月の📌が全部出る。
6. 1件開いて作業区分を「現場作業」に変えて保存 → 📌が外れ、一覧から消える（確定化）。

### ④ 倉庫
7. 倉庫作業を1件登録（代表者を選んで）→ 事務タブ「集計を更新」。
8. スプレッドシート「元請別請求集計_フィルタ用」に `現場名＝倉庫作業` の行が人別＋合計で出る。
9. オートフィルタで「現場名＝倉庫作業」→ 倉庫だけ抽出できる。
10. 「会社名（元請名）」で実在元請を選ぶ → 倉庫行（空欄）は出ない（請求に混ざらない）。
11. 通常の「元請別請求集計」シートには倉庫が出ないことを確認。
