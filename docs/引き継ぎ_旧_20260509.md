# 予定管理アプリ 引継書

**最終更新**: 2026-05-09

---

## 1. プロジェクト概要

建設現場向けの日報管理・粗利管理 Web アプリ。

- **フロントエンド**: HTML + JavaScript（GitHub Pages 配信）
- **バックエンド**: Google Apps Script（GAS）
- **データ保管**: Google スプレッドシート
- **対象**: 複数会社（グローライズ / 和信カインド / ビジネスサポート / ラーテル / GRHD / GRミツマ）
- **用途**: 日報入力、勤怠集計、工番管理、事業部別売上按分、粗利計算

## 5. 主な機能

### ユーザー向け（index.html）
- 日報入力（単日/期間指定）、カレンダー表示、空き確認、現場別管理
- モード切替: 🏭倉庫 / 🌙夜勤 / 🏖️休み / 📌予定
- 工番事業部セレクタ（グローライズで新規現場登録時、代表者事業部デフォルト）
- 現場名重複チェック（類似候補との紐付け確認）
- 異常値検出（出勤≥退勤 → 送信ブロック）
- ❓使い方ボタン（右下フローティング）
- 動的タイトル（`?c=会社名` → ホーム画面追加時「会社名予定表」）
- 会社インジケーター（画面上部に「🏢 会社名」表示）

### 管理者向け（admin.html）
- 全機能＋全社切替＋編集・削除
- 事務タブ: 月別集計 / CSVダウンロード / マスタ管理 / 集計更新 / アーカイブ / 工番修正
- 工番修正モーダル: 事業部変更（再採番）と売上入力
- 職人単価管理（粗利計算の元データ）

### 🔒 社長専用カレンダー（president.html）
- **完全独立**：日報データとは別の「社長予定」シートに保存。他の管理者・職人からは見えない
- PIN認証（`1203`）：sessionStorage で同タブ内のみ記憶、🔒ボタンで即時ロック
- ダーク基調の専用UI（テーマカラー#1a1a2e）。既存アプリと別ブランドで意図的に区別
- カレンダー表示（月切替、当日ハイライト、予定チップを色付きで表示）
- 一覧タブ（昇順/降順切替）
- FAB「＋」または日付タップ→「この日に追加」で新規予定
- 8色カラー × 8カテゴリ（商談/社内会議/来客/現場/出張/会食/プライベート/その他）
- 詳細画面から編集・削除
- `<meta name="robots" content="noindex,nofollow">` で検索エンジン除外
- **GAS側は全アクションでPINチェック**（フロントを迂回しても認証必須）

### GAS バックエンド（gas.js）
- 日報CRUD（add/update/delete）+ LockService による同時更新保護
- 工番自動採番: `[事業部]-[年度下2桁]-[連番3桁]` 形式（例: `ICT-26-001`）
- 年度は4月始まり、事業部×年度ごとの独立連番
- 倉庫モードは工番を発番しない
- 集計生成: 会社別 / 月別 / 月別確認表 / 元請別請求 / 事業部別按分
- 粗利計算: 原価 = 人工×単価、粗利 = 売上 − 原価
- 事業部不明フォールバック: 職人マスタに事業部ない→現場マスタの事業部で按分
- 自動トリガー: 日次集計（6時）/ 月次アーカイブ（1日3時）
- 操作ログ記録（add/update/delete/archive/reassign等）
- 工番一括バックフィル関数 `backfillJobNos()`
- **読み（フリガナ）自動生成**: Groq API 連携、`GROQ_API_KEY` 設定時のみ有効
- 読みバックフィル関数 `backfillAllYomi()`

## 6. 主な関数（gas.js）

### 公開エンドポイント
- `doGet(e)` — 日報・職人・元請マスタを返す
- `doPost(e)` — action別の処理

### doPost アクション一覧
| action | 処理 |
|---|---|
| add | 日報追加 |
| update | 日報更新（旧ID削除→新行追加） |
| delete | 日報削除 |
| archive | 手動アーカイブ |
| summarize | 集計再生成 |
| get_sheet | 指定シートの内容取得（CSV用） |
| add_member / remove_member | 職人追加/削除 |
| update_member_division | 職人の事業部更新 |
| update_member_rate | 職人の単価更新 |
| add_genba / remove_genba | 元請追加/削除 |
| update_site_revenue | 現場の売上入力 |
| reassign_jobno | 工番事業部の変更＋再採番 |
| 🔒 pres_list | 社長予定の一覧取得（PIN必須） |
| 🔒 pres_add | 社長予定の追加（PIN必須） |
| 🔒 pres_update | 社長予定の更新（PIN必須） |
| 🔒 pres_delete | 社長予定の削除（PIN必須） |

### メンテナンス関数（GAS エディタから手動実行）
| 関数 | 用途 |
|---|---|
| `setupAllTriggers()` | 日次集計＋月次アーカイブのトリガーを一括登録 |
| `dailySummary()` | 集計の手動実行（トリガーからも呼ばれる） |
| `autoArchive()` | アーカイブの手動実行（トリガーからも呼ばれる） |
| `backfillJobNos()` | 既存の日報・アーカイブから工番未設定の行に工番を一括付与 |
| `backfillAllYomi()` | 元請マスタ/現場マスタの「読み」空欄を Groq で埋める |
| `getOrCreatePresSheet_(ss)` | 「社長予定」シートを取得（無ければ作成→hideSheet）|

## 7. 初回セットアップ手順

### GAS 側
1. `deploy/gas/gas.js` を Google Apps Script エディタに貼付 → 保存
2. ウェブアプリとしてデプロイ（全員アクセス可）
3. 発行URLを `index.html` / `admin.html` の `GAS_URL` 定数に反映
4. `setupAllTriggers` を一度実行（トリガー登録）
5. （任意）スクリプトプロパティに `GROQ_API_KEY` を設定 → 読み自動生成が有効化

### GitHub Pages 側
1. `deploy/github-pages/` 内の全ファイルをリポジトリにアップロード
2. Settings → Pages → Source: `main` → Save
3. URL 確認

### マスタ初期設定
1. 管理画面 → 事務 → 職人を管理 → 全員登録（事業部・単価）
2. 元請マスタに元請を登録

## 10. 今後の開発・メンテナンス指針

### ファイル更新時の反映
- **gas.js 更新時**: GAS エディタに貼り直し → デプロイ更新。URL が変わったら `GAS_URL` 定数も更新
- **HTML/静的ファイル更新時**: `deploy/github-pages/` を GitHub Pages に push

### GAS URL の更新方法
1. 新しいURLを取得
2. `deploy/github-pages/index.html` と `admin.html` の `GAS_URL` 定数を置換
3. GitHub に push

### トラブルシューティング
| 症状 | 原因 | 対処 |
|---|---|---|
| 工番が空欄 | 代表者の事業部が未設定 | 職人管理で割当 |
| 粗利がマイナス | 売上未入力 / 単価が高すぎ | 工番修正で確認 |
| 原価が0 | 単価が未設定 | 職人管理で単価入力 |
| 「他の人が更新中」 | 同時書き込み | 数秒待って再送信 |
| 起動が遅い | 日報データ蓄積 | アーカイブ実行 |
| 集計が古い | 集計ボタン未押下 | 事務→集計を更新 |
| 事業部が「不明」になる | 応援職人（他社所属）| 現場マスタの事業部で按分される仕組み済み |

## 14. 🔒 社長専用カレンダー 運用メモ

### セキュリティ特性

| 項目 | 設計 | 限界 |
|---|---|---|
| URL秘匿 | `president.html` は links.html に非掲載 / `noindex,nofollow` | URLを知る人にはPIN画面までは到達可能 |
| PIN認証 | フロント/GAS両側でチェック | PINは平文で HTML/GAS ソースに記載 → ソースを読まれると判明 |
| データ分離 | 専用シート + シート非表示 | スプレッドシート所有者・編集者権限を持つ人は表示可能 |
| データ操作 | 必ず `president.html` 経由 | スプレッドシート直接編集は IDの整合性が崩れるため厳禁 |

### より強固にしたい場合の追加策（現在は未実装）

1. PINを長くする（10桁以上 / 記号混じり）
2. URLパラメータ方式（`?key=長いランダム文字列` + PIN の二重）
3. GAS側でアクセスログ記録
4. 別の GAS プロジェクト＋別スプレッドシートに完全分離

### 動作確認手順（メンテ時）

```powershell
$url = "https://script.google.com/macros/s/AKfycbxp2eUcpIjCj0ZWyAPPD9m3egJrKdWmXRK2AVnFrmBm4iO1QHCk-FZEH5LFFv7OloqcjQ/exec"
# 正しいPIN
Invoke-RestMethod -Uri $url -Method Post -ContentType "text/plain" -Body '{"action":"pres_list","pin":"1203"}'
# 間違ったPIN（拒否されることを確認）

```

### トラブル対処

| 症状 | 原因 | 対処 |
|---|---|---|
| PIN画面で解除できない | PIN不一致 / Caps Lock | 数字4桁を慎重に入力 |
| 解除後 "読み込み失敗: 認証に失敗しました" | フロントとGAS間でPIN不一致 | president.html と gas.js の PIN 定数が一致しているか確認 |
| 予定が反映されない | GAS側未デプロイ | Apps Script で「デプロイ → デプロイを管理 → 編集 → 新バージョン」 |
| シートが見当たらない | hideSheet で非表示 | スプレッドシート左下☰アイコン → 「社長予定」 |
| 社長予定が空になっている | 直接シート編集でID消失 | 操作ログにも残らないため復旧困難。直接編集禁止を徹底 |
