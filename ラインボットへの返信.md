## 実装した4つのアクション

依頼書の通り実装。すべて `logOperation_` で操作ログにも記録されます。

### A. `vehicle_res_add` — 予約追加

リクエスト:
```json
{
  "action": "vehicle_res_add",
  "token": "車両予約用トークン1234",
  "event": {
    "reservation_id": "R-20260511-103045",
    "vehicle_name": "ハイエース（白）",
    "plate": "なにわ432そ8800",
    "company": "グローライズ",
    "user_name": "向",
    "user_line_id": "U8a07d211e...",
    "start_dt": "2026-05-12T10:00:00+09:00",
    "end_dt_planned": "2026-05-12T17:00:00+09:00",
    "end_dt_actual": "",
    "destination": "大阪本社",
    "status": "予約",
    "memo": ""
  }
}
```

レスポンス:
```json
{ "status": "ok", "id": "R-20260511-103045" }
```

`status` を省略すると "予約" がデフォルト。

### B. `vehicle_res_update` — 予約更新（部分更新OK）

リクエスト:
```json
{
  "action": "vehicle_res_update",
  "token": "車両予約用トークン1234",
  "event": {
    "reservation_id": "R-20260511-103045",
    "status": "返却済",
    "end_dt_actual": "2026-05-12T16:45:00+09:00"
  }
}
```

レスポンス:
```json
{ "status": "ok", "updated": "R-20260511-103045" }
```

`event` に含まれるフィールドだけ更新。他は変更されない。更新日時は自動更新。

対象が見つからない場合: `{ "status": "error", "message": "対象が見つかりませんでした" }`

### C. `vehicle_res_delete` — キャンセル（論理削除）

リクエスト:
```json
{
  "action": "vehicle_res_delete",
  "token": "車両予約用トークン1234",
  "id": "R-20260511-103045"
}
```

レスポンス:
```json
{ "status": "ok", "cancelled": "R-20260511-103045" }
```

行は削除せず、状態を「キャンセル」に更新します（履歴を残すため）。

### D. `vehicle_res_list` — 予約一覧取得

リクエスト:
```json
{
  "action": "vehicle_res_list",
  "token": "車両予約用トークン1234"
}
```

レスポンス:
```json
{
  "status": "ok",
  "rows": [
    {
      "予約ID": "R-20260511-103045",
      "車両名": "ハイエース（白）",
      "ナンバー": "なにわ432そ8800",
      "所有会社": "グローライズ",
      "使用者氏名": "向",
      "使用者LINE_ID": "U8a07d211e...",
      "開始日時": "2026-05-12T10:00:00+09:00",
      "返却予定日時": "2026-05-12T17:00:00+09:00",
      "実返却日時": "",
      "行先": "大阪本社",
      "状態": "予約",
      "備考": "",
      "登録日時": "2026-05-11T10:30:45+09:00",
      "更新日時": "2026-05-11T10:30:45+09:00"
    }
  ]
}
```

Date型は ISO 8601 (`yyyy-MM-dd'T'HH:mm:ssXXX`) で返却。

## エラーレスポンス共通

- トークン不一致: `{ "status": "error", "message": "認証失敗" }`
- 予約ID未指定: `{ "status": "error", "message": "予約IDが指定されていません" }`
- 対象なし: `{ "status": "error", "message": "対象が見つかりませんでした" }`

## シート構造（参考）

スプレッドシートに「車両予約」シートが自動作成され、ヘッダは以下の14列：

| # | カラム名 | 内容 |
|---|---|---|
| 1 | 予約ID | R-20260511-103045 形式 |
| 2 | 車両名 | 例: ハイエース（白） |
| 3 | ナンバー | 例: なにわ432そ8800 |
| 4 | 所有会社 | GRHD / グローライズ / 等 |
| 5 | 使用者氏名 | 社員DB上の氏名 |
| 6 | 使用者LINE_ID | U... 形式 |
| 7 | 開始日時 | ISO 8601 |
| 8 | 返却予定日時 | ISO 8601 |
| 9 | 実返却日時 | 空 or ISO 8601 |
| 10 | 行先 | 例: 大阪本社 |
| 11 | 状態 | 予約 / 利用中 / 返却済 / 期限超過 / キャンセル |
| 12 | 備考 | 任意のメモ |
| 13 | 登録日時 | GAS 自動採番 |
| 14 | 更新日時 | GAS 自動採番 |

シートはスプレッドシートの先頭タブ群とは別に追加されています。初回POSTで自動作成されるので、手動でシートを作る必要はありません。
